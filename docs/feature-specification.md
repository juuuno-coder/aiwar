# AI 대전 - 기능 명세서

## 📋 개요
이 문서는 AI 대전 게임의 모든 기능을 우선순위별로 정리한 명세서입니다.

---

## 🎯 개발 우선순위

### MVP (Minimum Viable Product) - Phase 1
게임의 핵심 기능만 구현하여 플레이 가능한 프로토타입 제작

### Phase 2
게임성 향상 및 콘텐츠 확장

### Phase 3
소셜 기능 및 고급 기능 추가

---

## 🔥 Phase 1: MVP (필수 기능)

### 1. 유저 시스템
**우선순위**: 🔴 최고

#### 1.1 계정 관리
- [ ] 회원가입 (이메일 또는 소셜 로그인)
- [ ] 로그인 / 로그아웃
- [ ] 비밀번호 재설정
- [ ] 프로필 정보 (닉네임, 아바타)

#### 1.2 게임 데이터 저장
- [ ] 유저별 보유 카드 목록
- [ ] 유저별 AI 군단 슬롯 정보
- [ ] 게임 진행도 (스토리 챕터)
- [ ] 보유 화폐 (데이터 코인, 연구 포인트)

**기술 요구사항**:
- 데이터베이스: PostgreSQL 또는 MongoDB
- 인증: JWT 또는 세션 기반
- 비밀번호 암호화: bcrypt

---

### 2. 카드 시스템
**우선순위**: 🔴 최고

#### 2.1 카드 데이터 구조
- [ ] 카드 ID (고유 식별자)
- [ ] AI 군단 (Gemini, ChatGPT, Claude 등)
- [ ] 주력 분야 (텍스트, 이미지, 영상, 음악, 코드)
- [ ] 유닛 이미지 (Nova Canvas Pro로 생성)
- [ ] 등급 (커먼, 레어, 에픽, 레전더리)
- [ ] 능력치 (창의성, 정확성, 속도, 안정성, 윤리성)
- [ ] 총 전투력 (능력치 합산)
- [ ] 레벨 (1~10)
- [ ] 경험치

#### 2.2 카드 표시
- [ ] 카드 UI 디자인 (앞면)
- [ ] 카드 뒷면 디자인
- [ ] 카드 상세 정보 모달
- [ ] 등급별 카드 테두리 효과
- [ ] 카드 애니메이션 (뒤집기, 글로우 효과)

**기술 요구사항**:
- JSON 기반 카드 데이터 관리
- CSS 애니메이션 또는 Canvas API

### 4. 유닛 생성 시스템
**우선순위**: 🔴 최고

#### 4.1 자동 생성
- [ ] 타이머 기반 자동 생성 (30분 주기)
- [ ] 생성 알림 (새 유닛 도착)
- [ ] 생성된 유닛 수령 (인벤토리로 이동)

#### 4.2 확률 시스템
- [ ] 등급별 확률 적용 (커먼 60%, 레어 30%, 에픽 8%, 레전더리 2%)
- [ ] 능력치 랜덤 생성 (등급별 범위 내)
- [ ] 가챠 애니메이션 (카드 뽑기 연출)

#### 4.3 인벤토리
- [ ] 보유 카드 목록 표시
- [ ] 필터링 (등급, AI 군단, 주력 분야)
- [ ] 정렬 (전투력, 등급, 획득 순)
- [ ] 카드 상세 보기
- [ ] 최대 보유 개수 제한 (100장)

**기술 요구사항**:
- 서버 시간 기반 타이머
- 랜덤 생성 알고리즘

### 6. 대전 시스템 (PvE)
**우선순위**: 🔴 최고

#### 6.1 덱 구성
- [ ] 5장의 카드 선택
- [ ] 덱 저장 / 불러오기
- [ ] 여러 덱 프리셋 저장 (최대 3개)

#### 6.2 대전 진행
- [ ] 턴제 배틀 시스템
- [ ] 각 턴마다 카드 1장씩 공개
- [ ] 능력치 비교 및 승패 판정
- [ ] 5라운드 중 3라운드 선승제
- [ ] 대전 결과 화면 (승리/패배)

#### 6.3 장르별 대전
- [ ] 대전 시작 전 장르 표시
- [ ] 장르별 능력치 가중치 적용
  - 창작 대결: 창의성 x2
  - 정확도 대결: 정확성 x2
  - 속도 대결: 속도 x2
  - 윤리 대결: 윤리성 x2
  - 종합 대결: 모든 능력치 균등

#### 6.4 대전 UI
- [ ] 대전 화면 레이아웃
- [ ] 카드 공개 애니메이션
- [ ] 능력치 비교 연출
- [ ] 승패 판정 애니메이션
- [ ] 라운드 진행 표시 (1/5, 2/5 등)

#### 6.5 보상
- [ ] 승리 시 데이터 코인 획득
- [ ] 사용한 카드 경험치 획득
- [ ] 스토리 진행도 업데이트

**기술 요구사항**:
- 대전 로직 엔진
- 애니메이션 시퀀스 관리

- **데이터베이스**: PostgreSQL 또는 MongoDB
